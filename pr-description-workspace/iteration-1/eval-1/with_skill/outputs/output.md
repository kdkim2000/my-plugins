## Summary

로그인 페이지에 이메일 형식 검증과 비밀번호 최소 길이(8자) 검증을 추가합니다. 각 유효성 검사 실패 시 사용자에게 명확하고 구체적인 에러 메시지를 개별적으로 표시해, 입력 오류를 빠르게 파악하고 수정할 수 있도록 UX를 개선합니다.

## Changes

- **이메일 유효성 검사**: 이메일 형식(정규식 또는 표준 검증 로직)에 맞지 않을 경우 에러 표시
- **비밀번호 최소 길이 검사**: 8자 미만 입력 시 별도 에러 메시지 표시
- **에러 메시지 분리**: 이메일/비밀번호 각각 독립적인 에러 메시지로 사용자에게 안내

## How to Test

1. 로그인 페이지로 이동
2. 이메일 필드에 형식이 잘못된 값 입력 (예: `notanemail`, `user@`, `@domain.com`)
3. 이메일 관련 에러 메시지가 표시되는지 확인
4. 이메일을 올바른 형식으로 수정 후, 비밀번호 필드에 7자 이하 값 입력 (예: `abc123`)
5. 비밀번호 관련 에러 메시지(최소 8자)가 표시되는지 확인
6. 두 필드 모두 유효한 값 입력 후 로그인 시도 → 에러 없이 정상 진행되는지 확인

## Notes for Reviewers

에러 메시지 문구와 표시 위치(인라인 vs. 상단 요약 등)가 기존 디자인 시스템과 일관성을 유지하는지 확인 부탁드립니다. 비밀번호 정책이 추후 변경될 경우(예: 특수문자 필수, 최대 길이 등) 확장이 용이한 구조인지도 함께 검토해 주세요.
