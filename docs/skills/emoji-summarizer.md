# emoji-summarizer

텍스트 / 문장 / 상황 설명을 **이모지 3~5개와 한 줄 설명**으로 압축해 주는 스킬입니다.

- 소스: [`plugins/my-skills/skills/emoji-summarizer/SKILL.md`](../../plugins/my-skills/skills/emoji-summarizer/SKILL.md)
- 자동 트리거: 예 (description 키워드 매칭)

## 어떤 입력에서 트리거되는가

다음과 같은 한국어 표현이 포함된 입력에서 자동으로 호출됩니다.

- "이모지로 요약"
- "이모지로 표현"
- "이모지 버전"
- "이모지로 설명"

이 키워드가 명시적으로 들어 있지 않더라도, 문맥상 "텍스트를 이모지로 압축해 보여줘" 라는 의도가 분명하면 모델이 자동 선택합니다.

명시적으로 호출하려면:

```
/my-skills:emoji-summarizer 오늘 비 와서 집에서 책 읽었어
```

## 출력 규칙

1. 이모지는 **3~5개** (매우 짧은 입력은 1~2개도 허용)
2. 시간순 / 인과순으로 배치
3. 이모지 다음 줄에 **한 줄 설명** (가볍고 친근한 톤)
4. 응답에는 코드펜스(```) 를 사용하지 않음

## 예시

**입력:** "오늘 비가 와서 집에서 따뜻한 커피 마시며 책 읽었어"
**출력:**
🌧️ ☕ 📖
비 오는 날 집에서 커피와 책, 완벽한 하루!

**입력:** "발표가 너무 긴장됐는데 결국 잘 끝났다"
**출력:**
😰 🎤 ✅
긴장했지만 해냈다 — 고생했어요!

**입력:** "야근하다가 버그를 잡고 새벽에 배포 성공"
**출력:**
🌙 🐛 🚀
야근 끝에 버그 잡고 배포 완료, 수고했어요!

**입력:** "친구랑 다퉜는데 오늘 화해했어"
**출력:**
💬 🤝 🌤️
오해를 풀고 다시 맑아진 사이.

## 엣지 케이스

| 상황 | 처리 |
| --- | --- |
| 매우 짧은 입력 (한 단어 / 한 구문) | 이모지 1~2개로 축약 |
| 모호 / 중립적인 내용 | 💭 🤔 같은 중립 이모지 사용 |
| 부정 / 슬픈 감정 | 위로하는 가벼운 톤 (과장된 슬픔 이모지 남용 금지) |
| 개인정보 / 민감 정보 포함 | 그 부분은 요약에서 제외, 일반화된 이모지로 표현 |
| 이모지로 표현 어려운 추상 개념 | 가장 가까운 비유 이모지 + 한 줄 설명에서 보강 |

## 잘 안 될 때 점검 포인트

1. **자동 호출 안 됨** — `description` 의 트리거 키워드가 누락되지 않았는지 확인. 슬래시 명령(`/my-skills:emoji-summarizer ...`)으로 직접 호출해 동작 자체는 살아 있는지 확인.
2. **응답에 ``` 가 끼어 나옴** — SKILL.md 본문의 "출력 형식" 섹션이 코드펜스 안에 들어 있어 모델이 그대로 흉내냄. "예시일 뿐, 실제 응답에는 코드펜스를 쓰지 말 것" 한 줄을 명시.
3. **이모지가 너무 많거나 적음** — 규칙 섹션의 "3~5개" 가이드를 본문 윗부분으로 끌어올리거나, 예시를 같은 패턴(이모지 3개)으로 통일.

## 커스터마이징 아이디어

- **언어 추가:** 영어 입력에 대한 예시 1~2개를 추가하면 다국어 입력에도 안정적으로 동작.
- **출력 형식 변경:** "이모지 → 한 줄 설명" 외에 "이모지 + 해시태그" 같은 변형은 새 스킬 폴더로 분리(예: `emoji-tagger`)하는 것을 권장. 동일 스킬 안에서 분기 시키면 트리거 정확도가 떨어집니다.
- **자동 호출 차단:** 슬래시 명령으로만 사용하고 싶다면 frontmatter 에 `disable-model-invocation: true` 추가.
